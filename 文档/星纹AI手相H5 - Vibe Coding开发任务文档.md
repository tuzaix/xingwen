# 星纹AI手相H5 \- Vibe Coding开发任务文档

---

## 1\. 整体开发规划

### 1\.1 项目总览

**项目目标**：打造基于 AI 多模态模型的手相面相分析 H5 服务，通过上传手部照片，结合用户信息生成专业的命理分析报告。

**核心功能矩阵**：

- 用户端 H5：照片上传 → 信息填写 → 卡密验证 → AI 分析 → 报告生成与分享

- 后端服务：卡密管理、图片预处理、AI 多模态分析、报告存储

- 管理后台：卡密生成与管理、报告查询、数据统计

- AI 层：手相特征提取 \+ 命理分析双 Agent 架构

**预期交付时间**：

- 开发周期：2 周（10 个工作日）

- 联调测试：3 天

- 上线部署：1 天

### 1\.2 技术栈确认

|层级|技术选型|版本要求|
|---|---|---|
|**前端 H5**|Vue 3 \+ Vite \+ Tailwind CSS \+ TypeScript|Vue 3\.4\+, Vite 5\.0\+, Tailwind 3\.4\+|
|**管理后台**|Vue 3 \+ Element Plus \+ TypeScript|Element Plus 2\.5\+|
|**后端 API**|FastAPI \+ Python 3\.11|FastAPI 0\.109\+, Python 3\.11\.8|
|**数据库**|MySQL 8\.0 \+ Redis 7|MySQL 8\.0\.36\+, Redis 7\.2\+|
|**AI 模型**|GPT\-4o / Gemini 3\.1 / Claude 3|OpenAI GPT\-4o, Google Gemini 3\.1 Pro|

### 1\.3 开发环境准备

**环境配置要求**：

- Node\.js \&gt;= 20\.10\.0

- Python \&gt;= 3\.11\.0

- Docker \&amp; Docker Compose

- MySQL 8\.0 \+ Redis 7（本地开发可使用 Docker）

**工具链**：

- VS Code \+ Volar \+ ESLint \+ Prettier

- Postman / Apifox（接口调试）

- TablePlus / DBeaver（数据库管理）

- Git \+ GitHub / GitLab

**代码规范**：

- 前端：ESLint \+ Prettier，遵循 Vue 3 Composition API 规范

- 后端：PEP 8 \+ Black 格式化，类型注解完整

- 提交规范：Conventional Commits（feat/fix/docs/chore）

## 2\. 任务拆解原则（Vibe Coding 模式）

### 2\.1 任务粒度标准

- **每个任务 2\-4 小时可独立完成**

- 单个开发者可在半个工作日内完成并提交 PR

- 避免 \&\#34;大任务\&\#34; 拆分为多个小任务并行

### 2\.2 验收标准要求

- **可量化**：有明确的完成判断标准

- **可验证**：可通过自动化测试或人工操作验证

- **可交付**：完成后可独立部署 / 演示

### 2\.3 依赖关系管理

- 依赖链不超过 3 层

- 无循环依赖

- P0 任务无外部依赖或仅依赖最基础的初始化任务

### 2\.4 优先级定义

- **P0**：阻塞性任务，必须最先完成，影响整个项目进度

- **P1**：核心功能任务，影响主要业务流程

- **P2**：优化增强任务，可延后或并行开发

## 8\. Vibe Coding 开发建议

### 每日开发节奏

- **上午**：领取 1 个任务，专注开发（9:00\-12:00）

- **下午**：完成第 2 个任务，提交 PR（14:00\-17:00）

- **晚间**：Code Review \+ 合并代码

### PR 规范

- 单 PR 对应单任务 ID

- PR 标题格式：`\[FE\-001\] 项目初始化与基础配置`

- 描述包含：完成内容、测试截图、验收点

- Reviewer：1 人即可，快速通过

### 快速验证

- 每个任务完成后，在开发环境快速验证

- 无需等待完整测试，符合验收标准即可合并

- 问题在后续迭代中快速修复
