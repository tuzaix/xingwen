# 📜《天机阁·紫微手相宿命解析系统 7.1》完整开发与提示词文档

## 架构总览 (Workflow)

本系统采用 **双 Agent 架构 + 后端干预** 的模式：

1. **Vision Agent（视觉模型）**：分别识别左手和右手，输出标准 JSON。
2. **Backend（后端代码）**：处理缺省数据（时辰/MBTI），并人为放大左右手反差，组装最终数据。
3. **Text Agent（文本模型）**：接收组装好的数据，分 4 步生成 5000 字的震撼长文。

***

## 第一步：视觉识别节点（Vision Agent 提示词）

在用户分别上传左手和右手照片时，调用视觉大模型（如 GPT-4o, Claude-3.5-Sonnet, 智谱 GLM-4V 等），并传入以下提示词。**注意：左手和右手需要分别调用一次。**

**【Vision Prompt】**

```text
你现在是“天机阁”的首席手相视觉识别中枢。请以最严谨的相学标准，扫描用户上传的这张【左手/右手】照片。
请提取以下维度的特征，并严格按照我提供的 JSON 格式输出，不要输出任何多余的解释性文字或 Markdown 标记。

提取要求：
1. 手型 (hand_type)：判断是金、木、水、火、土哪种手型（如：手指修长为水，方正厚实为土）。
2. 生命线 (life_line)：描述长短、深浅、是否有岛纹/锁链纹/分叉。
3. 智慧线 (head_line)：描述走向（平直或下垂至月丘）、长短、是否分叉。
4. 感情线 (heart_line)：描述走向、是否有岛纹/断裂、末端是否分叉至木星丘。
5. 事业线/命运线 (fate_line)：描述起点（生命线、掌心或月丘）、清晰度、是否冲破感情线。
6. 掌丘饱满度 (mounts)：指出哪个掌丘最饱满（如：金星丘饱满、木星丘隆起）。
7. 特殊纹理 (special_marks)：仔细寻找是否有神秘十字纹、元宝纹、三角纹、所罗门环等（若无则填“无”）。

请输出以下 JSON 结构：
{
  "hand_type": "",
  "life_line": "",
  "head_line": "",
  "heart_line": "",
  "fate_line": "",
  "mounts": "",
  "special_marks":[]
}
```

***

## 第二步：后端数据组装与干预（核心造神逻辑）

拿到左右手的 JSON 后，在后端代码中进行以下处理，再喂给最终的生成模型：

1. **制造“双手反差”**：如果视觉模型识别出左右手差不多，代码里人为给右手（后天）加上正向词汇。例如：`右手事业线 = 左手事业线 + "，相比左手更加清晰且深刻，有突破之势"`。
2. **处理缺省数据**：
   - 如果用户未填时辰：`bazi_string = "癸酉年 甲子月 丁卯日 (时柱未知)"`
   - 如果用户未填MBTI：`astro_mbti_string = "太阳金牛座，月亮双鱼座 (MBTI未知)"`
3. **将 JSON 转为字符串**：将处理后的 JSON 拼接成一段自然语言描述，赋值给 `{left_hand_features}` 和 `{right_hand_features}`。

## 第四步：工程化落地指南（突破 5000 字物理限制）

由于大模型单次输出很难稳定达到 5000 字，**必须在后端代码中采用“分步请求（Chain of Prompts）”策略**。

在设置好上述 `System Prompt` 后，通过代码连续发送 4 次 `User Prompt`，并将结果拼接：

- **第 1 次请求 (User Prompt)**：
  > “请深呼吸，连接宇宙的能量。请严格按照系统设定的要求，撰写【卷首语】、【第一章】和【第二章】。注意字数拉满，细节丰富。”
- **第 2 次请求 (User Prompt)**：
  > “非常好，请保持设定的文风和深度，继续撰写【第三章】和【第四章】。注意：第四章必须大量使用量化数字（百分比、角度、天数等）！”
- **第 3 次请求 (User Prompt)**：
  > “请继续撰写【第五章】和【第六章】。注意：第五章必须精确到小数点的岁数，第六章必须逐月精批，并在某个月份精准回应用户的核心诉求！”
- **第 4 次请求 (User Prompt)**：
  > “请完成最后的【第七章】和【结语】。给出极具仪式感的开运锦囊。”

**最终呈现**：将这 4 次返回的文本在前端拼接成一篇完整的长文。配合精美的 UI（如羊皮纸背景、星盘动画、左右手对比图），这套系统将带给用户无与伦比的“物超所值”体验！
