## 1\. 产品定位与核心功能模块

### 1\.1 产品定位

**星纹 AI 手相管理后台**是为 \&\#34;星纹\&\#34;AI 手相 H5 产品打造的独立运营管理系统，面向产品运营人员、技术管理员和商务合作方，提供卡密生命周期管理、用户报告管理、数据统计分析、系统参数配置等核心能力，实现产品运营的数字化、精细化管理。

**核心价值**：

- 实现卡密全生命周期的可视化管理

- 提供用户行为和产品使用数据的多维度分析

- 支持系统参数的动态配置与模型切换

- 记录所有管理员操作行为，保障系统安全可追溯

### 1\.2 核心功能模块

#### 1\.2\.1 卡密管理模块

**功能定位**：卡密是 \&\#34;星纹\&\#34;AI 手相产品的核心付费载体，本模块实现卡密从生成、分发、核销到作废的全生命周期管理。

**核心功能**：

|功能点|详细说明|
|---|---|
|**批量生成**|支持按批次批量生成卡密，可设置：<br>\- 生成数量（1\-10000 张 / 批次）<br>\- 卡密类型（月卡 / 季卡 / 年卡 / 次卡）<br>\- 有效期（绝对时间 / 相对时间）<br>\- 备注信息|
|**卡密导出**|支持 Excel 格式导出，包含：<br>\- 卡密编号、明文密码<br>\- 生成时间、有效期<br>\- 批次号、状态、核销时间<br>\- 导出时自动添加数字水印|
|**条件查询**|多维度组合查询：<br>\- 卡密编号 / 批次号精确查询<br>\- 状态筛选（未使用 / 已使用 / 已过期 / 已作废）<br>\- 时间范围筛选<br>\- 类型筛选|
|**状态管理**|支持单张 / 批量状态变更：<br>\- 手动作废（作废原因必填）<br>\- 延期操作<br>\- 重置使用状态|
|**批次管理**|按批次维度统计：<br>\- 批次生成数量<br>\- 已核销数量、核销率<br>\- 作废数量|

**卡密生成算法**：

- 卡密格式：16 位大写字母数字组合，分为 4 组，每组 4 位

- 示例：`XW2A\-3B7C\-9D2F\-8E4H`

- 前缀校验：前 2 位固定为 \&\#34;XW\&\#34; 标识

- 校验位：最后 1 位为校验位，采用 Luhn 算法

#### 1\.2\.2 报告管理模块

**功能定位**：管理所有用户生成的 AI 手相分析报告，支持查询、查看和统计分析。

**核心功能**：

|功能点|详细说明|
|---|---|
|**报告查询**|多维度查询条件：<br>\- 用户 ID / 报告 ID 精确查询<br>\- 报告状态（生成中 / 已完成 / 生成失败）<br>\- 时间范围筛选<br>\- 卡密关联查询|
|**详情查看**|报告完整信息展示：<br>\- 用户基本信息（脱敏展示）<br>\- 手相图片缩略图<br>\- 分析结果全文<br>\- 生成耗时、使用模型<br>\- 关联卡密信息|
|**报告重生成**|对失败报告支持手动触发重生成<br>可选择不同模型进行重新分析|
|**报告导出**|支持单份报告 PDF 导出<br>支持批量导出报告统计数据|
|**异常监控**|自动识别生成失败、超时的报告<br>异常报告告警通知|

#### 1\.2\.3 用户数据统计模块

**功能定位**：提供产品运营数据的多维度统计分析，支撑运营决策。

**核心指标**：

|指标类别|具体指标|计算口径|
|---|---|---|
|**用户活跃**|日活用户（DAU）<br>周活用户（WAU）<br>月活用户（MAU）<br>用户留存率|DAU：当日访问 H5 的独立用户数<br>留存：次日 / 7 日 / 30 日留存|
|**使用量**|报告生成总量<br>日均生成量<br>峰值时段分布<br>人均生成次数|统计成功生成的报告数量|
|**卡密核销**|累计核销数量<br>日核销数量<br>核销率<br>各类型卡密核销占比|核销率 = 已核销卡密数 / 已发放卡密数|
|**转化分析**|访问到付费转化率<br>卡密类型分布<br>用户地域分布|转化漏斗分析|

**数据看板**：

- 核心指标实时展示

- 趋势图表（近 7 天 / 30 天）

- 地域分布热力图

- 时段分布柱状图

#### 1\.2\.4 系统配置模块

**功能定位**：管理系统运行参数、AI 模型配置、API 密钥等核心配置。

**核心功能**：

|配置类别|配置项|说明|
|---|---|---|
|**模型配置**|默认分析模型<br>备选模型列表<br>模型调用参数<br>降级策略|支持 GPT\-4o、Gemini 3 等多模型切换<br>可配置各模型的权重和调用阈值|
|**API 密钥管理**|OpenAI API Key<br>Google API Key<br>阿里云 OSS 密钥<br>密钥轮换记录|密钥加密存储<br>支持密钥版本管理和轮换|
|**业务参数**|单用户日生成上限<br>报告有效期<br>卡密默认有效期<br>限流阈值|参数实时生效，无需重启服务|
|**通知配置**|告警接收人<br>告警阈值配置<br>通知渠道（邮件 / 钉钉 / 企业微信）|异常事件自动告警|

### 3\.3 权限控制设计

#### 3\.3\.1 JWT 认证流程

```Plain
┌─────────┐    用户名密码     ┌─────────┐
│  浏览器  │ ───────────────> │  后端   │
│         │ <─────────────── │         │
└─────────┘    JWT Token      └─────────┘
     │                            │
     │ 后续请求携带 Header:        │
     │  Authorization: Bearer xxx │
     └────────────────────────────┘
```

**JWT Token 结构**：

- Header：算法 \+ 类型

- Payload：

    ```json
    {
      "sub": "admin_123",
      "username": "admin",
      "role": "super_admin",
      "permissions": ["card:*", "report:*", "config:*"],
      "exp": 1714888800
    }
    ```

- Signature：HS256 签名

**Token 配置**：

- Access Token 有效期：2 小时

- Refresh Token 有效期：7 天

- 支持主动注销（Redis 黑名单）

#### 3\.3\.2 RBAC 权限矩阵

|权限项|超级管理员|运营管理员|技术管理员|
